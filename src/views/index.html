项目中所犯的BUG

ios中a标签绑定事件bug

描素猴年马月项目中在一个a标签绑定了click事件 $(document).on('click','ele'.function);在安卓上没有异常，在ios上 事件失效

原因ios a标签的click事件有bug

解决代码改为$(document).on('touchstart','ele',function);但是会发生击穿事件，在document上我绑定了一个click事件， 让弹框影藏。touch后会冒泡到document上。需要取消冒泡。加入以下代码 function(e){ e=e||window.event; e.stopPropagation;return false;}

补充关于stopPropagation的一些小问题

  • ie9及ie9以上阻止冒泡时使用e.stopPropagation
  • 低版本ie使用e.cancelBubble
  • jquery中阻止冒泡使用stopPropagation或者return false;前者已经兼容了事件冒泡的
  • return false;和stopPropagation的区别
    • return false 包含阻止冒泡和组织默认行为，例如a标签的跳转
    • stopPropagation仅仅只是组织冒泡而已

样式书写规范不当导致的ie样式显示错乱的bug

描素A+流量撮合平台中border-top在其他浏览器中能显示，但是在ie中硬是检查不出来

原因css中样式书写错误border-top:1px solid#333333;样式书写不规范，没有空格隔开，在谷歌中没有异样，但是在ie中就错误了

解决空格去掉

容易混淆的知识点

offsetHeight,style.height,$().height(),clientHeight()几者的区别

  • 1、obj.offsetHeight可以获取写在样式文件里的属性值，而obj.style.height只能获取行内属性
  • 2、obj.offsetHeight返回的是一个数值，而obj.style.height返回的是一个字符串，单位是“px”
  • 3、obj.offsetHeight是只读，而obj.style.height是可读写
  • 4、如果没有为元素设置高度，obj.offsetHeight会根据内容获取高度值，而obj.style.height会返回undefind
  • 5、jQuery里我们使用$(obj).height()来获取元素的高度，（$(obj).css('height')返回的是一个带有单位的字符串）。 与其他三个（$(obj).height()、$(obj).css('height')、obj.style.height） 不同的是，obj.offsetHeight的高度就是height+padding,其他的设置同css一样，根据box-sizing不同而不同
  • 6、clientHeight+border为offsetHeight
  • 7、注意$().css("height")和style.height获取到的是字符窜，其他获取到的int类型
